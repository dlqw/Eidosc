# Eidosc 0.6.0-alpha.1 dot Namespace syntax

- Replaced qualified-name `::` and slash module paths with one dot-based Namespace surface while retaining `::` exclusively for declaration bindings.
- Made commas the only ADT constructor separator and added targeted migration diagnostics for removed `::` qualifiers and `|` constructor separators.
- Added project-aware import normalization for uppercase Namespace roots and lowercase dependency package aliases followed by uppercase modules.
- Extended `eidosc migrate syntax` to migrate Eidos 0.5 source and manifests to 0.6 without rewriting declaration bindings.
- Updated formatter, diagnostics, HIR/MIR identity displays, LSP semantic tokens, standard-library sources, fixtures, and generated binding tests for the new syntax.
- Adopted the Eidos owl mark and horizontal EIDOS wordmark in the public English and Chinese READMEs.
