# Eidosup 0.4.0-alpha.1

## Components and immutable variants

- Added strict component manifests with exclusive file ownership,
  dependency/conflict validation, exact artifact identity, executable metadata,
  and signed/checksum-bound loading.
- Added `component list/add/remove` and immutable variants whose identity binds
  the distribution manifest, profile, selected files, explicit components, and
  explicit targets.
- Upgraded install manifests to schema 3 and toolchain state to schema 3, with
  verified migration of earlier supported state and unsupported legacy
  aggregate installs reported as unmanaged.

## Profiles, targets, and projects

- Added manifest-defined `minimal`, `default`, and `complete` profiles plus
  `set profile` and `show profile`.
- Preserved explicit component and target additions across profile changes and
  updates; project auto-sync adds missing requirements without deleting
  existing explicit selections.
- Added `target list/add/remove` with separate compiler, runtime, linker
  command, and external SDK readiness.
- Made channel rollback skip composition variants of the current release and
  made release uninstall remove all retained variants transactionally.

## Offline documentation and automation

- Added the `eidos-docs` component and `eidosup doc [topic]` with strict
  version matching, path confinement, browser, path-only, and JSON modes.
- Added `--installed` filtering for component and target lists, completion
  entries, and published JSON schemas for component, target, mutation, and docs
  results.
- Extended Eidosc and Eidosup release workflows with component-manifest
  generation, executable ownership verification, candidate and post-publication
  anonymous clean installs on all six native RIDs, cross-target object smoke
  tests, host-native executable generation, and complete uninstall validation.
