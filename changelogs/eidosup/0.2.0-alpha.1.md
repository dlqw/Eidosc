# Eidosup 0.2.0-alpha.1

## Immutable toolchain state

- Added immutable toolchain IDs derived from Eidosc version, host RID, release metadata, and the verified asset digest.
- Added schema 2 install manifests binding the complete installed file set to the immutable toolchain identity.
- Added schema 1 toolchain state for installed toolchains, selectors, defaults, activation history, transactions, and unmanaged directories.
- Added separate state locking, write-through temporary files, atomic replacement, backups, idempotent initialization, and manifest-based recovery from corrupt state.
- Refused unknown or future state schemas and stopped importing the legacy version-only install layout.
- Extended `eidosup doctor` to compare state with verified immutable manifests without repairing state implicitly.

## Stable shims and resolution

- Added an owned stable `bin/eidosup` manager and `bin/eidosc` multi-call shim, using a hard link when supported and a durable copy fallback.
- Added transactional pair replacement, ownership metadata, conflict refusal, and idempotent shim repair.
- Activated the first verified install as the global default and allowed channel updates to move that default without changing `PATH`.
- Verified state, host, install manifest, complete payload, compiler, and runtime before activation.
- Preserved arguments, standard streams, working directory, console signal group, child environment, and compiler exit codes across proxy execution.
- Changed environment configuration to use only the stable bin directory and removed legacy version-bound `EIDOSC_HOME` and `EIDOS_RUNTIME_PATH` settings.
- Enabled ReadyToRun for self-contained release binaries and added Windows/Linux startup-overhead gates with a 200 ms incremental median limit.

## Toolchain management

- Added verified `toolchain install`, `list`, and transactional `uninstall` commands for channel and exact-version selectors.
- Added explicit global `default` selection and clearing without changing `PATH`.
- Added `update` and non-mutating `check` commands for installed channels or requested toolchain specifications.
- Added structured `show`, `show active-toolchain`, `show home`, and bootstrap-profile reporting.
- Added explicit `run` and `which` commands using the same verified resolver and process-forwarding contract as the stable shim.
- Recorded channel activation history and added rollback to retained verified immutable installations.
- Serialized lifecycle commits with a per-root management lock and recovered interrupted uninstall transactions before later mutations.
- Resolved management roots from `EIDOS_HOME` unless an explicit `--install-root` override is supplied.
- Preserved an explicitly cleared default across later install and update operations and prevented exact-version selectors from rebinding to a different release identity.

## Verification

- Added machine-readable output, dry-run planning, CLI contract coverage, concurrent two-version installation tests, and uninstall fault-injection recovery tests.
- Updated clean-install release gates to exercise toolchain management, resolve the installed compiler through state, and compile a pinned public tutorial fixture.
