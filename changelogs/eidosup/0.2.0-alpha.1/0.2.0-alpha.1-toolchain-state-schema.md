# Eidosup 0.2.0-alpha.1 toolchain state schema

- Added immutable toolchain IDs derived from Eidosc version, host RID, release metadata, and the verified asset digest.
- Added schema 2 install manifests that bind the complete installed file set to the immutable toolchain identity.
- Added schema 1 toolchain state covering installed toolchains, selectors, defaults, activation history, transactions, and unmanaged directories.
- Added separate state locking, write-through temporary files, atomic replacement, backups, idempotent initialization, and manifest-based recovery from corrupt state.
- Refused to overwrite unknown or future state schemas.
- Stopped importing the legacy version-only install layout and added explicit manual cleanup guidance.
- Extended `eidosup doctor` to compare state with verified immutable manifests without repairing state implicitly.
- Updated clean-install release gates to resolve the installed compiler through state and compile a pinned public tutorial fixture.
