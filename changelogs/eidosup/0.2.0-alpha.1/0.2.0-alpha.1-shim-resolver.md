# Eidosup 0.2.0-alpha.1 stable shim and resolver

- Added an owned stable `bin/eidosup` manager and `bin/eidosc` multi-call shim,
  using a hard link when supported and a durable copy fallback.
- Added transactional pair replacement, ownership metadata, conflict refusal,
  and idempotent shim repair.
- Activated the first verified install as the global default and allowed channel
  selector updates to move that default without changing PATH.
- Added activation-time state, host, install-manifest, complete payload file,
  compiler, and runtime verification while allowing the compiler-owned grammar
  cache generated beside existing Eidosc releases.
- Preserved arguments, standard streams, working directory, console signal
  group, child environment, and compiler exit codes across proxy execution.
- Changed environment configuration to use only the stable bin directory and to
  remove legacy version-bound `EIDOSC_HOME` and `EIDOS_RUNTIME_PATH` settings.
- Added Windows/Linux clean-install shim compilation, exit-code, and median
  startup-overhead release gates with a 200 ms incremental baseline.
- Enabled ReadyToRun for self-contained release binaries to keep full selected
  payload verification inside the proxy startup baseline.
