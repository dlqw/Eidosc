# Eidosup 0.1.0-alpha.2

## Distribution and release selection

- Added a public GitHub release-source boundary with environment-only authentication for private custom sources.
- Added strict SemVer 2.0.0 parsing and deterministic `stable` and `preview` channel resolution.
- Added stable structured errors, redacted verbose diagnostics, machine-readable error output, documented exit codes, and actionable `doctor` results.

## Verified atomic installation

- Required `SHA256SUMS` coverage and verified asset size and SHA-256 digest before installation.
- Added resumable bounded downloads, finite transient retries, and a revalidated content-addressed artifact cache.
- Rejected unsafe ZIP paths, links, conflicting entries, special files, and excessive archive expansion.
- Recorded and rehashed the complete installed file set in an installation manifest.
- Serialized setup operations and added staging, atomic replacement, rollback copies, and durable transaction-journal recovery.
- Kept setup dry runs free of install, download, cache, lock, and transaction filesystem changes.

## LLVM dependency providers

- Defined LLVM/Clang 18 through 22 and the required `clang` and `llvm-ar` commands as the supported dependency contract.
- Separated dependency probing, provider planning, execution, and post-install verification.
- Added deterministic non-interactive plans for Windows, Linux, and macOS package managers, including explicit elevation handling.
- Refused to replace an installed incompatible LLVM version implicitly.
- Reported combined LLVM compatibility through `eidosup doctor` and stable dependency error codes.

## Release candidate gates

- Added deterministic checksum and release metadata generation for Eidosc and Eidosup assets.
- Added complete RID-set, digest, size, NuGet package, and safe ZIP structure verification before publication.
- Added isolated Windows x64 and Linux x64 installation tests that run `eidosc info` and compile a tutorial smoke source.
- Changed release workflows so tags and public prereleases are created only after candidate verification succeeds.
