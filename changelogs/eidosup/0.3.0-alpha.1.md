# Eidosup 0.3.0-alpha.1

## Project selection

- add strict `eidos-toolchain.toml` project pins, ancestor search, canonical
  directory overrides, `EIDOSUP_TOOLCHAIN`, and shim `+toolchain` selection;
- define deterministic explicit/environment/nearest-project-or-override/default
  precedence and report the exact selector source;
- validate project language versions against toolchain `compatibility.json`
  metadata without changing `eidos.toml`;
- add configurable interactive/non-interactive missing-toolchain auto-install.

## Local development toolchains

- add `toolchain link/unlink` for read-only `custom:<name>` compiler builds;
- revalidate the external executable/runtime layout on every activation and
  guarantee that update, uninstall, and self-uninstall never delete the linked
  directory.

## Sources, offline operation, and trust

- add persistent direct sources and named multi-mirror source groups with
  trust-before-priority fallback;
- preserve signed-metadata, integrity, and state failures when every equally
  trusted mirror fails instead of misclassifying them as network failures;
- add HTTPS/file signed-index and complete offline-bundle providers, safe bundle
  import/export, special-file/path/expansion defenses, content-addressed storage,
  and bundle-atomic bounded LRU cache cleanup;
- add Ed25519 signed metadata verification with an official pinned root,
  expiration, monotonic rollback protection, repeatable transition/commit key
  rotation with revocation, asset size and SHA-256 binding, and enterprise
  bootstrap keys;
- add the release metadata signer, official GitHub secret integration, and
  Eidosc release workflow generation/reverification of `eidosup-index.json`;
- reject credentials in persisted index URLs and rely on standard proxy/no-proxy
  environment handling without logging credentials.

## Eidosup lifecycle and CLI

- add verified `self update --check-only`, atomic Unix replacement, delayed
  Windows replacement, downgrade prevention, candidate-aware stale-stage
  recovery, and non-recursive explicit lifecycle execution;
- add ownership-checked `self uninstall --yes [--keep-toolchains]` with symmetric
  PATH/profile cleanup and external/unmanaged path preservation;
- add validated `settings.toml` for default host, self-update, auto-install, and
  active source policy, with no-write dry runs for settings and source-group
  mutations;
- extend `doctor` with explicit validation of linked external custom toolchains;
- add deterministic Bash, Fish, Zsh, and PowerShell completion generation;
- add global quiet/color options, ANSI diagnostic policy, stderr-only download
  progress for JSON safety, string-valued JSON enums, and successful broken-pipe
  handling.

## State and distribution

- migrate toolchain state schema 1 to schema 2 atomically, adding custom links
  and directory overrides while preserving immutable installs and activation
  history;
- include `compatibility.json` in newly published Eidosc host bundles;
- embed native Ed25519 support in the self-contained single-file release.
