# Eidosc 0.5.0-alpha.1 capability-constrained Build host

- Added manifest parsing, validation, normalization, and canonical writing for `[build]`, `fileInputs`, registered environment variables, output roots, and `[[build.tools]]`.
- Added the compiler-built-in typed `Build` schema with separate filesystem, environment, process, and emit capabilities that are unavailable to ordinary pure comptime.
- Added canonical `BuildGraph` materialization and diagnostics for cycles, duplicate outputs, undeclared inputs, missing dependency edges, output containment, producer mismatches, and host/target misuse.
- Added stable input-directory expansion, content/environment/executable path-and-content hashing, physical input/program/output disjointness, host-filesystem path identity, sanitized tool environments, deterministic topological execution, declared-output verification, and output-validated BuildGraph caching.
- Added generated-source import-root integration and included build program, graph, input, environment, tool, host/target, and output fingerprints in full-build artifact identity.
- Added `eidosc build --emit-build-graph` and `--trace-build` plus unit/integration coverage for capability denial, manifest round trips, execution/cache restore, input tracking, and invalid graphs.
