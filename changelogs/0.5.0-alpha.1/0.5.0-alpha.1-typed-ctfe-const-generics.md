# Eidosc 0.5.0-alpha.1 typed CTFE and const generics

- Added a closed typed compile-time value model with stable canonical serialization, recursive pure evaluation, typed HIR reification, and cache round trips for scalar, String, tuple, list, and ADT values.
- Added ordered type/value/effect-row generic argument domains and value-level const generics across parsing, naming, Types, HIR/MIR, specialization, layout, LLVM name mangling, coherence, diagnostics, payloads, fingerprints, and LSP semantic details.
- Added const-generic support for ADTs, type aliases, constructors, functions, traits, `@impl(...)`, and named instances, including symbolic propagation and concrete specialization.
- Advanced the supported Eidos language version and embedded standard-library manifest to `0.5.0-alpha.1` without changing the independent Eidos Std product version.
- Upgraded Types payload schema to 13 and MIR payload schema to 7 for the new generic-domain and value-identity metadata.
