# Eidosc 0.4.0-alpha.3

## Toolchain distribution

- Published strict host-specific toolchain manifests that bind Eidosc, Eidos
  language compatibility, independently versioned Std/runtime/bindgen
  components, profiles, targets, exact file ownership, artifact sizes, and
  SHA-256 digests.
- Added the external `stdlib/` publish layout and runtime lookup through
  `EIDOS_STDLIB_PATH`, `EIDOS_RUNTIME_PATH`, and
  `EIDOS_TARGETS_PATH`.
- Added version-matched offline documentation, bindgen, host runtime, and all
  published cross-target runtimes to verified Eidosc host bundles.
- Added deterministic generation and verification of six component manifests
  before release signing and clean installation.

## Compiler targets

- Added the `aarch64-pc-windows-msvc` target and `win-arm64` aliases.
- Added stable runtime-identifier mapping for all six published Windows, Linux,
  and macOS x64/ARM64 targets.
- Resolved installed target runtimes before the host runtime and isolated
  runtime-resolution caches by target RID and configured target root.

## Verification

- Extended all six native Windows, Linux, and macOS x64/ARM64 candidate gates
  with external Std inspection, component/profile/target transactions, offline
  documentation, opposite-architecture LLVM IR/object generation, host-native
  executable generation, immutable-variant checks, and complete uninstall.
- Added a post-publication anonymous six-RID install gate so release visibility
  and native startup cannot be satisfied only by CI credentials or cross-builds.
- Calibrated the shim startup regression gate per native RID and made every job
  report direct, shim, incremental, and upper-bound medians.
