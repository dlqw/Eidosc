# Fixed-point metaprogramming, stable slots, and coherence increment

- Added stage-scoped fixed-point scheduling with deterministic ordering, dependency edges, transitive query invalidation, canonical graph cycle detection, and a bounded non-convergence diagnostic.
- Added `meta.slot_from` and `meta.with_slot` so generators can derive stable output identities from declaration, type, field, constructor, and canonical scalar keys without coupling identity to collection order.
- Split generation-slot identity from payload identity, making identical slot/payload regeneration a no-op while rejecting conflicting reuse atomically.
- Preserved explicit slot identity and generated provenance through live-state, symbol-table, HIR, CLI, IDE, and generated-document payloads.
- Added prospective coherence validation for generated instances and function `impl` clauses, including generic heads and nominal or instance-member replacement/removal, without leaking speculative diagnostics into the fixed-point scheduler.
- Prevented convention-based function implementations from synthesizing invalid implementations for methodless marker traits.
