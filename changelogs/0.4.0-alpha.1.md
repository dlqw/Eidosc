# 0.4.0-alpha.1

- Added nominal marker effects declared with `Name :: effect;` and effect-row parameters declared with `E: effects`.
- Functions and higher-order callbacks propagate authorization through `need`; fixed tags and row variables can be combined.
- Removed ability operations, handlers, `with`, `resume`, CPS lowering, and runtime effect dispatch.
- Updated qualified paths and module re-exports so effects and ordinary functions are exported independently.
- Made effect rows part of generalized and cross-module type summaries and added conservative effect-aware MIR optimization summaries.
- Updated all standard-library callback APIs to preserve callback effects and kept borrow authorization independent from effects.
- Preserved native FFI symbols through effect-only generic specialization and restored canonical qualified calls in mixed module MIR cache builds.


## Function overload resolution
- Added ordinary same-scope function overload groups with type-directed dispatch.
- Added overload selection for direct calls, qualified calls, method calls, infix calls, and pipe calls.
- Added expected-function-type selection for bare overload references, including concrete-over-generic specificity ranking and diagnostics when the reference remains ambiguous.
- Added declaration diagnostics for duplicate parameter signatures and value/function overload collisions.
- Hardened MIR function-name fallback maps so overloaded source names do not collapse to the last declaration when `SymbolId` identity is missing.
- Hardened `@ffi` overload validation so unqualified external symbols are checked for duplicate bindings just like `lib/symbol` bindings.
- Added intrinsic registry lookup by compiler intrinsic name plus normalized signature key for future intrinsic overload sets.
- Added trait method overload validation and named instance signature-key matching for overloaded trait methods.
- Added query-pipeline regression coverage for overload additions, deletions, imported overload invalidation, and import-order candidate determinism.
- Added IDE semantic snapshot overload groups, overload-aware completion metadata, and LSP hover overload lists for resolved overload members.
- Kept trait, ability, and instance implementation methods in their existing dispatch domains instead of treating them as ordinary overload duplicates.
- Added regression coverage for overload dispatch, duplicate signatures, alpha-equivalent generic signatures, expected-type overload references, qualified overloads, pipe/infix/method calls, FFI overload metadata, intrinsic signature registry lookup, overloaded trait method implementation, and instance-domain exclusion.


## 0.3.3-alpha.3: Incremental cache and parallel builds

- Added an exact-input cache fast path for resolved, typed, LLVM IR, and native builds, keyed by compiler build identity, source/import content, target/backend settings, FFI inputs, and output-independent build configuration.
- Added cache payload SHA-256 verification, atomic artifact writes, cross-process cache locking, capacity-based pruning, temporary/orphan artifact collection, and `cache status`, `cache clean`, `--no-cache`, and `--cache-max-mib` controls.
- Added real mixed module execution for Namer, Types, HIR, and MIR: changed modules compile new phase payloads while unaffected modules restore validated payloads, with deterministic merge/remap behavior and bounded same-layer parallelism.
- Changed stale, missing, or dependency-drifted restore candidates to compile the affected module instead of blocking the stage and forcing a whole-program fallback.
- Localized Namer, Types, HIR, MIR, symbol-table, module-registry, and AST payloads to their owning modules, avoiding per-module copies of whole-program state.
- Added stable cross-session symbol/type/function/constructor identities for payload remapping, MIR ownership, runtime constructor layouts, LLVM lowering, and destructor synthesis.
- Propagated `--jobs` through module task execution, source hashing, LLVM codegen, FFI/native compilation, and runtime archive construction while preserving deterministic output ordering.
- Removed duplicate raw typed-semantic facts, advanced the typed semantic snapshot schema to v2, and retained canonical facts plus current-build symbol/type ids for stable invalidation and diagnostics.
- Reduced JSON hash allocation by reusing serializer configuration and hashing direct UTF-8 serialization without an intermediate UTF-16 string.
- A local 200-function cache fixture now occupies about 14.36 MiB; measured typed builds were about 7.22 s cold, 5.18 s with module-artifact restore, and 0.52 s through the exact-cache path.
- The cold `snake-gui-types` profile now measures about 1.18 s with Types at about 283 ms / 26.24 MiB, compared with `develop@1e07c2f` at about 28.5 s and 6.03 GiB of Types allocation under the same profile case.

## Unreleased - List Slice Patterns

## Added

- List patterns now support one rest marker in the middle of the pattern, matching the common C# list-pattern and Rust slice-pattern shape:
  - `[head, ..middle, last]`
  - `[..init, last]`
  - `[first, .., last]`
- The rest binding continues to use Eidos spelling, `..name` or `.._`, and still allows at most one rest marker per list pattern.

## Changed

- `[a, ..rest, b]` is now valid and binds `rest` to the middle slice.
- Duplicate rest markers such as `[a, ..rest, ..tail]` remain parser errors.
- Complex rest subpatterns such as `..Some(x)` are rejected; use prefix/suffix element patterns around `..rest` instead.

## Verified

- Parser, advanced pattern parsing, parser recovery, list-pattern coverage regression, and native LLVM list-pattern smoke tests pass for the new middle-rest and suffix forms.
