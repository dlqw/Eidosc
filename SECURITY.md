# Security Policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
vulnerability reporting for this repository. If that is unavailable, email
`quxinyue.dev@gmail.com` with a description, affected versions, reproduction,
and impact assessment.

Reports will be acknowledged after receipt and coordinated before public
disclosure. Please avoid including credentials or unrelated personal data.

## Supported versions

Eidosc is currently prerelease software. Security fixes are released in the
next available prerelease rather than backported to every older prerelease.
